## ¿Qué es Git?

Git es un sistema de control de versiones que permite a los programadores...
Trabajar en equipo de una manera ordenada y eficiente, permitiendo que cada integrante del equipo pueda trabajar y compartir sus avances sin afectar el trabajo de los otros participantes mediante las ramas, además guarda todas las modificaciones hechas para poder regresar a versiones anteriores si se necesita y se puede observar que cambios se hicieron, cuando y quien los hizo.

## Por qué necesitamos Git

Sin Git, cuando trabajamos en equipo...
Se puede sobreescribir el codigo si dos personas trabajan en lo mismo, además sería dificil saber que cambios hizo cada persona, en conclusión sería todo más desorganizado.

## Los 3 conceptos clave de Git

1. **Repositorio:** Es en donde se guardan todos los datos de el proyecto
2. **Commit:** Es un punto de guardado de el archivo en ese momento con un mensaje de que fue lo que cambió
3. **Rama:** es una parte de código independiente dentro del proyecto que permite desarrollar funciones sin cambiar el código principal.

## Los comandos que aprendí
git log --all --oneline: Muestra una lista resumida de todos los commits realizados por todos los colaboradores.

git init: Inicializa un nuevo repositorio Git en la carpeta actual.

git add README.md: Prepara el archivo README.md para ser guardado en un commit.

git commit -m "mensaje": Guarda los cambios preparados en el historial con un mensaje descriptivo.

git remote add origin [URL]: Conecta tu repositorio local con un repositorio remoto en GitHub.

git push: Sube tus commits locales al servidor de GitHub.

git pull: Descarga y fusiona los cambios más recientes desde GitHub a tu computadora.

git clone [URL]: Descarga una copia completa de un repositorio existente en GitHub a tu máquina local.

git push -u origin main: Sube tus cambios a GitHub y establece la conexión permanente entre la rama local y la remota.
